Agentic SDLC Dashboard (OAP team)

Overview

Project overview and summary

Goals

1

Accelerate Earliest Adoption

Deliver a minimal, tool-agnostic (orchestration/model/sandbox), production-safe agentic flow for operator teams on real work, without waiting for full platform maturity.

2

Serve Every SDLC Persona

Structure the flow so SEs, SSEs, and Architects each have explicit gates, artifacts, and approval points aligned to their responsibilities.

3

Shift Quality Left

Use planning-stage reviews and human gates to surface bugs, design flaws, and scope risks before code generation begins.

Challenges and Recommendations

Key challenges encountered and recommendations for improvement

Key Challenges

  • Alignment of agentic flow with human thoughts for ease of review and adaptation (for example, the review of tasks.md and code gen followed by PR should follow a modular approach instead of mass generation and one PR approach).
  • Mis-interpretation of Spec/contextual docs by LLMs/agents and delayed identification of issues.
  • Cross-repo governance and secured agentic flow.

Recommendations

  • Adaptation of OpenSpec Orchestration (customized) for agentic SDLC given the flexibility for customization and native integrations as well as alignment with human reviewer thought process.
  • Hybrid approach to building and refining the contextualization docs using ai-helpers agentic docs and Cursor. Ai-helpers generated certain sections which were more meaningful and aided in higher efficiency.
